# Решение тестовых задач на C++

---

## Структура проекта

* `task1_bitvector.cpp` — Класс битового вектора произвольной разрядности.
* `task2_weights.cpp` — Алгоритм выбора элемента на основе весовых коэффициентов.
* `task3_memory.cpp` — Поиск и анализ максимальных непрерывных подпоследовательностей запросов памяти.

---

## 🛠 Описание реализации задач

### Задача 1. Класс произвольного битового вектора (`BitVector`)
Реализован класс динамического битового вектора, оптимизированный по памяти (1 бит вектора занимает ровно 1 бит в памяти).


* **Конструктор**: Шаблонный, инициализирует вектор на основе указателя на переменную или массив **любого интегрального типа** (`uint8_t`, `uint16_t`, `int32_t` и т.д.). Нулевой разряд элемента массива соответствует нулевому разряду вектора. Лишние биты в последнем байте автоматически маскируются. Использование математических сдвигов гарантирует **платформонезависимость** (результат не зависит от Endianness процессора).
* **Работа с битовыми полями**: Поддерживаются методы записи и чтения произвольных битвекторов (`set_field` / `get_field`). Для полей малой разрядности реализованы **шаблонные** методы `set_field_small` / `get_field_small`, которые принимают и возвращают значения **любого интегрального типа** (с автоматической защитой от Undefined Behavior при сдвигах).
* **Копирование в массив**: Шаблонный метод `copy_to_array` позволяет безопасно выгружать данные в массив переменных любого интегрального типа с корректным расчетом необходимого количества элементов.
* **Строковое представление**: Метод `to_hex_string()` возвращает 16-ричное представление вектора. Старшие разряды выводятся слева (стандартный математический формат записи чисел, MSB first), а незначащие лидирующие нули автоматически отбрасываются.

### Задача 2. Выбор элемента по весам (`get_weighted_element`)
Реализован масштабируемый шаблонный алгоритм для выбора случайного элемента из массива $\{a_i\}$ на основе весов $\{w_i\}$ и вещественного числа $x \in [0, 1]$.

### Задача 3. Анализ непрерывных областей памяти (`build_sequences`)
Алгоритм запускает поиск только из «источников» — запросов, у которых нет стыкующихся по адресу предшественников, что гарантирует максимальность цепочек с самого начала. От каждого такого источника выполняется рекурсивный обход строго вперёд по совпадающим адресам памяти, а финальный путь фиксируется только при достижении тупика, когда цепочку больше невозможно продлить.

## 🚀 Сборка и запуск

Для сборки проекта вам понадобится компилятор с поддержкой **C++17** или выше (например, `GCC`, `Clang` или `MSVC`).

### Компиляция через терминал (на примере GCC

```bash
# Компиляция задачи 1 (Битвектор)
g++ -std=c++17 task1_bitvector.cpp -o task1

# Компиляция задачи 2 (Выбор по весам)
g++ -std=c++17 task2_weights.cpp -o task2

# Компиляция задачи 3 (Запросы памяти)
g++ -std=c++17 task3_memory.cpp -o task3
